# Key implementation details:

TokenStorageImpl delegates to src/auth/oauth.ts functions:
- saveTokens(email, tokens) -> writes ~/.emailcli/tokens/{email}.json
- getTokens(email) -> reads and parses that file, returns null if not found
- deleteTokens(email) -> removes the file (no error if doesn't exist)
- listAccounts() -> reads ~/.emailcli/tokens/ directory, returns .json filenames without extension
- refreshAccessToken(email) -> reads tokens, refreshes using oauth2Client.setCredentials, saves

ConfigImpl delegates to src/utils/config.ts functions:
- loadConfig() -> reads ~/.emailcli/config.json, creates with defaults if missing
- getConfigPath() -> returns ~/.emailcli/config.json

Note: oauth.ts uses os.homedir() to find the home directory. For testing, we need to either:
1. Mock os.homedir() to return temp directory
2. Or test at integration level without mocking (use actual ~/.emailcli but with unique test prefix)

We will use approach: set HOME to a temp directory in beforeEach and reset in afterEach.
