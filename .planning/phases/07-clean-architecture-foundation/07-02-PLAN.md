**CLI logic that needs to move into services (from cli.ts action callbacks):**

1. **resolveProvider()** - This is CLI logic that resolves account to provider. Stays in commands layer, not services.
2. **list command (lines 226-247):** Validates limit, calls provider.list(), outputs JSON
3. **status command (lines 254-266):** Calls provider.status(), outputs JSON
4. **folders command (lines 273-285):** Calls provider.listFolders(), outputs JSON
5. **read command (lines 294-312):** Calls provider.read() or provider.readThread(), outputs JSON
6. **search command (lines 325-345):** Validates limit, calls provider.search(), outputs JSON
7. **send command (lines 362-430):** Parses options, validates body, calls provider.send(), outputs JSON
8. **reply command (lines 441-469):** Parses options, calls provider.reply(), outputs JSON
9. **account add/list/remove commands:** Delegate to AccountService with OAuth flow

**Key constraint:** Services should NOT call console.log/console.error. They return data. Output formatting is the commands layer's responsibility (per Clean Architecture).
